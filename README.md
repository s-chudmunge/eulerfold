EulerFold is a technical learning engine designed to move developers from passive consumption to active mastery. Most learning platforms focus on completion certificates and video progress; EulerFold focuses on **Proof of Work**. It builds adaptive curriculums, validates technical submissions with AI, and tracks skill depth across a decentralized learning record.

### Why EulerFold?

Passive learning is a trap. Collecting bookmarks and watching tutorials creates an illusion of competence that collapses during real-world implementation. EulerFold bridges this gap by enforcing a "do-to-learn" cycle. Every roadmap is generated based on your specific goals or job descriptions, and every milestone requires a validated technical submission.

### Core Architecture

- **Adaptive Planning**: Week-by-week curriculums generated by Gemini 2.0/2.5 Pro, optimized for your current skill gap.
- **Job Decoded**: Transform raw Job Descriptions into personalized bridging roadmaps.
- **Study Planner**: Dynamic task scheduling based on your daily intensity (Casual, Balanced, or Intense).
- **Research Lab**: Direct interface for mastering technical papers with deep-dive analysis and recall testing.
- **Proof of Work**: Manual upload of code, PDFs, or images evaluated by a technical AI reviewer with zero-fluff feedback.
- **Earned Records**: Export your progress as a cryptographically verifiable PDF credential with a QR code.

### Tech Stack

EulerFold is built for scale and low-latency interaction:

- **Frontend**: Next.js (App Router), Tailwind CSS v4, Lucide Icons.
- **Backend**: FastAPI (Python 3.11+), Redis for caching, Async core.
- **AI**: Google Gemini (Native Async integration), custom technical evaluation prompts.
- **Infrastructure**: Supabase (PostgreSQL, Auth, Storage) with composite indexing for high-density data.
- **Email**: Resend for transactional alerts and credential delivery.

### Setup

#### Backend
```bash
cd backend
python -m venv venv && source venv/bin/activate
pip install -r app/requirements.txt
# Configure SUPABASE_URL, SUPABASE_KEY, and GEMINI_API_KEY in .env
uvicorn app.main:app --port 8080 --reload
```

#### Frontend
```bash
cd frontend
npm install
# Configure NEXT_PUBLIC_SUPABASE_URL and NEXT_PUBLIC_SUPABASE_ANON_KEY in .env.local
npm run dev
```
